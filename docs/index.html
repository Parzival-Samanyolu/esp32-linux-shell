ESP32 Linux Shell + Web Desktop

A Linux-style terminal and a full windowed web-desktop OS on a $6 microcontroller — with a camera, an SD card, three programming languages, GPIO control, games, and its own Wi-Fi hotspot.

Deneyap WROVER-E ESP32-D0WD-V3 · 240 MHz 8 MB PSRAM @ 80 MHz ESP-IDF · C · -O2
no router needed — the board hosts its own Wi-Fi. Join ESP32-Wifi, then:
telnet 192.168.4.1 2222
…or open the graphical desktop in a browser:
http://192.168.4.1/gui

$ whatis

An ESP-IDF firmware that turns the board into a networked, Linux-flavoured terminal. Connect with nc / telnet / PuTTY and it behaves like a real shell — files live on a real SD card, HTTP requests are real, and every system stat comes from live hardware APIs. Then it goes further: a camera, a browser file manager, a text editor, and embedded interpreters for Python, JavaScript and C.

🖥️

ESP32-OS web desktop

A full graphical desktop in the browser at /gui — draggable windows, a taskbar & start menu, and 10+ apps: files, terminal, editor, code runner, system monitor, GPIO panel, camera, games, paint and more.

📡

Own Wi-Fi hotspot + captive portal

The board broadcasts its own network (ESP32-Wifi · 192.168.4.1), so it works anywhere with no router. Join it and the desktop pops open automatically — like a hotel Wi-Fi page. It also joins a known Wi-Fi when in range.

🐚

Linux-style shell

ls, cat, cp, mv, grep, wc, head, tail, find, du, hexdump, df, wget, curl, ping, free, uptime, ifconfig, dmesg — plus history and a live htop (with temperature).

💡

GPIO & LED control

gpio read/write/blink, pwm, and onboard-LED effects — led breathe / pulse / rainbow — with a pin guard that protects the camera/SD pins.

📷

Camera (OV2640)

photo — 1600×1200 JPEG to SD. video — records an MJPEG .avi clip. stream — live MJPEG in your browser.

🎮

Games & fun

Snake, 2048, Tetris, Pong, Tic-Tac-Toe (perfect AI) & Minesweeper in the desktop — plus terminal toys: cmatrix, cowsay, fortune.

🔳

QR codes

qr prints a scannable QR right in the terminal (or on the desktop) — by default the hotspot login, so a phone can scan to connect.

💾

SD storage + editor

FAT32 over SPI with real files & timestamps, a browser file manager, and a full-screen nano editor over telnet.

🩺

Diagnostics

selftest (PSRAM span, SD speed, camera, GPIO, NTP, temp) and stress — CPU int/float throughput plus internal-RAM vs PSRAM bandwidth.

$ open /gui

A complete windowed desktop, served straight from the board and rendered in your browser — no app to install, no internet, works over the board's own hotspot. Everything is vanilla HTML/CSS/JS embedded in the firmware and gzip-compressed to ~14 KB.

$ run --lang

Three real interpreters embedded in the firmware. Write scripts to the SD card and run them by extension: run script.py · run app.js · run prog.c

python — pocketpy (Python 3 subset)
>>> print(2**64)
18446744073709551616
>>> [i*i for i in range(5)]
[0, 1, 4, 9, 16]
js — Duktape (ES5/ES2015)
js> [1,2,3].map(x => x*x)
1,4,9
js> JSON.stringify({ok:true})
{"ok":true}
cc — PicoC (interpreted C)
// hello.c
int main(){ printf("sum=%d\n",1+2+3); }
→ sum=6
These are compact embedded interpreters, not full CPython/Node/GCC — the ESP32 can't host a real compiler or JVM. They run genuine (subset) Python, JavaScript and C directly on the chip.

Build
PlatformIO
Notable internals: each network client runs in its own FreeRTOS task; large buffers use PSRAM; the Python interpreter's static data is relocated to PSRAM via a linker fragment; the JS engine's heap uses custom PSRAM allocators; and the C interpreter ships with a bespoke ESP-IDF platform layer.
Built on the Deneyap WROVER-E with ESP-IDF & PlatformIO.
